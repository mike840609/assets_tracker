# UI/UX Priority Framework

Recommended order when triaging UI/UX improvements:

1. **Core task clarity and completion**
   - Users should be able to complete their primary task quickly and confidently.
2. **Error prevention and recovery**
   - Add clear validation, actionable error messages, and visible system feedback.
3. **Performance and responsiveness**
   - Reduce waiting time, improve perceived speed, and avoid janky interactions.
4. **Accessibility**
   - Ensure keyboard navigation, readable contrast, and assistive-technology compatibility.
5. **Visual polish**
   - Refine spacing, typography, and animations after core usability is solid.

## Why this order?

The highest-impact UX improvements reduce friction in core journeys first. Visual polish is important, but only after users can reliably complete critical tasks.
